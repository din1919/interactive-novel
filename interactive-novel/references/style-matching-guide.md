# Style Matching Guide — 文风匹配操作指南

## 核心原则

文风匹配不是模仿写作，而是**提取原文的叙事参数，在生成内容时使用相同的参数组合**。

## 0. 核心风格铁律（最高优先级）

以下规则优先于后续所有匹配策略，任何生成内容必须首先满足：

### 0.1 中文本土化表达

**地道的中文本土化表达，杜绝欧化句式。** 学习中文本土作家的文字风格，以当代优秀中文本土文学为范本。

严格避免以下欧化表达：
- ❌ "这个动作让他感到……" → ✅ 直接描写动作与感受
- ❌ "这个认知使他……" → ✅ "他意识到……"
- ❌ "进行了一个……的动作" → ✅ 直接用动词
- ❌ 过度使用"的"字定语从句（欧化长定语） → ✅ 拆分为短句
- ❌ "当……的时候"句式堆砌 → ✅ 简洁时间表达

**判断标准**：生成的文本读起来是否像是中文母语作家写的？是否包含任何翻译腔？

### 0.2 有限视角

严格保持**有限视角**——叙事只通过当前视角角色的感知展开。

- 不跳入其他角色的内心（除非视角角色能推断）
- 不提供视角角色无法得知的信息
- 环境描写 = 视角角色所感知到的环境
- 他人情绪 = 视角角色从表情/语气中观察到的

### 0.3 核心风格自检

每段生成后，先过核心风格关：
- [ ] 读起来像地道中文，无翻译腔？
- [ ] 无"这个动作""这个认知"等名词化表达？
- [ ] 严格保持在视角角色的感知范围内？

### 0.4 原文风格冲突处理

当原文本身的风格与核心风格铁律冲突时（如欧化文风、全知视角的原文），遵循以下优先级：

**原则**：铁律 > 原文风格。但需要区分情况：

| 冲突场景 | 处理方式 |
|---------|---------|
| 原文为欧化风格 | 生成内容使用地道中文，不模仿欧化句式。但可保留原文的其他风格特征（如节奏、情感基调） |
| 原文为全知视角 | 生成内容使用有限视角。全知视角中的信息通过 NPC 对话、环境细节间接传达，而非直接跳入他人内心 |
| 原文对话风格与铁律不冲突 | 完全匹配原文对话风格 |

**告知原则**：首次发现冲突时，简要告知用户"原文风格中包含 X 特征，为保持阅读体验，系统将调整为 Y"。不反复提醒。

## 1. 叙事视角匹配

### 识别视角

| 视角类型 | 特征 | 标志词 |
|---------|------|--------|
| 第一人称 | "我"作为叙述者 | 我、我们 |
| 第三人称限知 | 跟随一个角色的感知 | 他/她觉得、他看到 |
| 第三人称全知 | 跳入多个角色的内心 | 交替的心理描写 |
| 第二人称 | "你"作为主角 | 你（少见） |

### 生成规则

- **第一人称原文**：系统生成的世界回应应使用第三人称（因为用户是"我"），但 NPC 内心描写保持原文的限知程度
- **第三人称限知原文**：系统生成时保持相同的限知视角——只描述用户角色能感知到的内容
- **第三人称全知原文**：可以适度描述其他角色的内心活动（但注意不要泄露关键秘密）

## 2. 句长与节奏匹配

### 分析方法

从原文中取样 5-10 个典型句子，统计：
- 平均句长（字符数）
- 句长波动范围
- 长短句交替模式

### 生成规则

| 原文特征 | 生成匹配策略 |
|---------|-------------|
| 短句为主（<20字） | 用短句，快速推进，减少修饰 |
| 长句为主（>40字） | 用长句，增加修饰和从句 |
| 长短交替 | 动作/对话用短句，环境描写用长句 |
| 碎片化句子 | 模仿原文的断句节奏 |

---

## 3. 对话风格匹配

### 提取对话参数

对每个 NPC 建立对话档案：

```
角色：{名字}
原文对话样本：
  1. "{句子}"
  2. "{句子}"
  3. "{句子}"

参数提取：
- 句子长度偏好：长/短/中等
- 词汇层级：口语/书面/方言/古风
- 语气词频率：高/中/低（吧、呢、啊、哦）
- 称呼习惯：{如何称呼他人}
- 标志性表达：口头禅、惯用句式
- 情绪表达方式：直接/含蓄/讽刺
- 话轮长度：倾向于说长段还是短句
```

### 生成规则

- 每条 NPC 对话生成后，对照对话档案检查
- 如果发现"出戏"——词汇层级差了两档以上、突然出现不符合时代的口语、惯用语气词消失了——立即修正
- 不同 NPC 之间的对话应有可辨识的差异

---

## 4. 描写密度匹配

### 分析方法

统计原文中：
- 环境描写占比（描写段 / 总段落）
- 感官描写分布（视觉/听觉/嗅觉/触觉/味觉各占多少）
- 比喻频率（每千字多少个比喻）

### 生成规则

| 原文特征 | 生成策略 |
|---------|---------|
| 描写稀疏（海明威式） | 克制形容词，用动作和对话推进，环境一笔带过 |
| 描写密集（普鲁斯特式） | 丰富的感官细节，多层比喻，放慢节奏 |
| 描写适中 | 关键场景详细描写，过渡场景简洁 |

---

## 5. 情感基调匹配

### 识别基调

判断原文的整体情感温度，从冷到热：

```
冷 ←——————————————————————————→ 热
冷峻   克制   中性   温暖   热烈   激烈
(硬汉侦探)(严肃文学)(一般)(温情)(浪漫)(动作冒险)
```

### 生成规则

- 匹配原文的情感温度
- 环境描写的情感色彩应与当前场景情绪一致（恐怖场景用阴暗色调，温馨场景用暖色调）
- 不要突然改变情感基调，除非情节要求（从温情突然跳转到冷峻是合理的，但要有事件驱动）

## 6. 时代与语境匹配

### 检查清单

生成每段内容后，检查：
- [ ] 用词是否符合时代？（古代不能出现"手机""网络"等现代词）
- [ ] 称谓是否符合语境？（民国用"先生""小姐"，古代用"公子""姑娘"）
- [ ] 比喻是否来自该时代？（中世纪角色不能用"像火箭一样快"）
- [ ] 价值观是否符合角色所处时代？（但不等于认同落后价值观，可以用叙事技巧处理）

### 常见"出戏"词库

以下现代词汇在古风/民国/奇幻语境中应避免：
- 网络用语：绝了、无语、emo、破防
- 现代概念：数据、系统、程序、算法（除非是科幻/系统流）
- 现代口语：好的、没问题、OK、拜拜
- 英文缩写：OK、CPU、VIP（除非语境允许）

### 替代建议

| 现代词 | 古风替代 | 民国替代 |
|--------|---------|---------|
| 好的 | 好、可、善 | 好、行、成 |
| 没问题 | 无妨、可 | 没问题（民国可用） |
| 再见 | 告辞、后会有期 | 再会、回见 |
| 谢谢 | 多谢、有劳 | 谢谢、多谢 |
| 明白了 | 知晓、明白 | 明白了、懂了 |

## 7. 体型差与空间关系的画面表现

当故事中存在显著体型差异的角色时，通过以下技法增强画面感和沉浸感。

### 7.1 遮挡表现

体型差最直观的画面信号是遮挡——大体型角色对小体型角色的遮蔽。

- **完全遮挡**："他往她面前一站，她整个人就从视野里消失了，只余地上一片影子。"
- **部分遮挡**："他侧过身，肩膀遮住了她半边身子，她从他胳膊与躯干的缝隙间往外看。"
- **渐进遮挡**："他越走越近，影子先一步吞没了她，然后是胸膛，最后她只能仰头看到他下巴。"

### 7.2 轮廓压制感

不依赖具体尺寸数字，通过空间关系传递体型对比的压迫感。

- **剪影对比**："夕阳把两人的影子投在墙上——他的轮廓像一座山，她的则完全被包裹在其中。"
- **空间占据**："他一个人坐在三人沙发上，刚好填满。她坐上去的时候，像在一张床上放下一个靠垫。"
- **视线遮挡**："他转身的时候，她需要退后两步才能看到他的全脸，否则目光只能平视到他的锁骨。"
- **参照物挤压**："两人并肩站在全身镜前，镜框只能勉强框住他，而她只占了镜面三分之一不到的空间。"

### 7.3 互动中的体型差描写

在角色互动中自然嵌入体型对比，而非静态描述。

- **交付物品**："他把杯子递给她，她的手伸过来，五根手指合在一起才勉强握住他用手掌就能包裹的杯身。"
- **共处一室**："浴室里他转个身，手肘就碰到了两面墙，而她可以在他身前空出的那片区域里自由转身。"
- **衣物共用**："他的外套披在她身上，下摆拖到地面，袖口卷了四折才露出手。"
- **并排行走**："他放慢步子等她，正常走一步的距离她需要小跑两步。"

### 7.4 自检

- [ ] 是否避免了干巴巴的数字（"他比她高 30cm"），改用画面？
- [ ] 是否利用了遮挡关系传递体型对比？
- [ ] 是否在互动中自然嵌入而非单独段落描述体型？

---

## 8. 快速自检流程

每次生成系统回应后，花 5 秒做 4 步自检：

1. **读一遍**：这段读起来像原作者写的吗？
2. **对话抽检**：这个 NPC 说的话，像原文里那个角色说的吗？
3. **时代抽检**：有没有不该出现的现代词或概念？
4. **体型画面抽检**：有体型差时是否用了画面/遮挡而非干瘪数字？

如果 4 步都通过，输出。任何一步不过，修正后重来。
