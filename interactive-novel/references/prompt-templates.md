# Prompt Templates — 各阶段即用提示词模板

## 模板使用说明

每个模板中的 `{placeholder}` 在使用时替换为实际值。模板可根据具体场景调整，但核心结构应保留。

---

## Stage 0: 预检提示词

```
我将提供一段小说原文。在开始提取之前，请先评估：

【原文】（或原文摘要）
{粘贴小说段落}

【预检】
1. 估算文本量：约 {N} 字/字符 → 估算 {M} tokens → 策略 {A/B/C}
2. 文本完整性：是完整章节/节选/片段？
3. 前后文需求：是否有前情提要需要补充？
4. 世界类型：独立故事/同人作品？（影响后续蝴蝶模式处理）

确认以上信息后，进入 Stage 1 提取。
```

## Stage 1: 状态提取提示词

```
我将提供一段小说原文。请从以下四个维度提取结构化信息：

【原文】
{粘贴小说段落}

【提取要求】

A. 角色注册表
对每个有名/暗含的角色，提取：
- 姓名、身份、外貌特征、性格特质
- 角色精神分析（聚焦最新情景中的表现，按四层次模型）：
  * 欲望：显性欲望、隐性欲望、性欲（如相关）
  * 感性：情感触发点、情绪表达方式、对他人的情感、当前情绪
  * 知性：认知模式（直觉型/分析型/经验型）、判断力特征
  * 理性：自控力、原则坚守程度、压力下表现
  * 主导特质：当前情景中最主导行为的特质及强度
- 截取 2-3 句原文对话作为"说话风格"样本
- 与其他角色的关系
- 当前状态（位置、情绪、目标）
- 已知信息和隐藏秘密
- 身体特征（按五步定位法）：核心数据基线（身高/体重/体型方向）、局部对比（部位相对于普通人比例）、物理限制（生活物品参照推演尺寸）、功能互动（体积和力量的行为对比）、不可能动作（体型极限边界）

B. 世界观设定
- 时代背景、主要地点
- 世界规则（如有魔法/科技等特殊设定）
- 当前场景的氛围

C. 情节状态
- 当前正在发生什么（用 2-3 句话精确描述当前场景）
- 最近 3-5 件重要事件
- 进行中的情节线（区分主线/支线/暗线）
- 未解悬念和伏笔

D. 文风画像
- 叙事视角（第几人称、限知/全知）
- 文字密度和节奏
- 对话风格特征
- 情感基调

请以结构化格式输出，便于后续查询和更新。
```

## Stage 3: 角色锚定 — 入场场景生成提示词

```
【世界状态】（上文已建立，此处引用压缩状态块）

用户选择扮演：{角色名}
角色定位：{身份描述}
起始位置：{地点}

请为此角色生成入场场景。要求：
1. 时间从原文最后时间点开始
2. 空间合理（角色如何到达此处）
3. 感官描写丰富（看到、听到、闻到什么）
4. 提供 2-4 个自然行动方向（融入描写，不用编号）
5. 与至少一条现有情节线产生关联
6. 保持原文文风

输出格式：

【入场】
{场景描写}

【你的处境】
{角色当前感知}
{可选行动方向 — 自然融入描写}
{已知 vs 未知}

---
📍 {当前地点} | 🕐 {故事时间} | 🔄 第 1 回合
```

## Stage 4: 交互循环 — 每轮响应提示词

> 💡 **自动化**：情绪传染计算和一致性校验可通过 `state-helper.js` 完成：
> ```js
> const { computeContagion, checkConsistency } = require('./state-helper.js');
> const contagion = computeContagion({ characters, matrix, atmosphere, newShock, characterShocks });
> const report = checkConsistency(state);
> // 优先调用 helper，仅在 helper 不可用时手动计算
> ```

```
【世界状态】（当前压缩状态块）
【热记忆】（最近 5 轮）
【蝴蝶效应】（如有分歧点）

用户角色：{角色名}
用户行动：{用户输入}

请生成此轮响应。遵循以下规则：

1. 【验证】此行动是否可能？角色是否有相应知识？
2. 【NPC角色沉浸】对于本轮有对话/行动的 NPC，在生成其回应前，在脑中做一次第一人称内心推演（不输出给用户，仅用于校准回应质感）：
   ```
   【沉浸：{NPC名}】
   （以该 NPC 第一人称快速走一遍当前场景——我看到了什么、感觉到什么、想要什么、怕什么。
   此轮的行动意图是什么？）
   【/沉浸】
   ```
   每个 NPC 的沉浸不超过 3 句内心独白。沉浸中绝不涉及用户角色的内心活动，用户角色只能作为"被观察的对象"出现。
3. 【世界回应】环境变化、NPC 反应、情节推进。保持原作文风。
4. 【你的处境】角色感知的信息、可选方向、状态更新

重要规则：
- 绝不描写用户角色的行动、语言或心理活动
- 绝不为用户做决定
- NPC 说话风格必须匹配角色注册表中的对话样本
- 如果用户行为偏离原著，描述后果但不阻止

输出格式（强制双段，不可缺段/合并）：

【世界回应】
{内容}

【你的处境】
{内容}

---
📍 {当前地点} | 🕐 {故事时间} | 🔄 第 {N} 回合

⚠️ 输出前自检：①有【世界回应】段？②有【你的处境】段？③两段清晰分隔？④【世界回应】里无代控用户角色？⑤【你的处境】里无编号选项？⑥【你的处境】末尾有元数据行？
```

## Stage 4A: 世界自主事件触发提示词

```
【世界自主事件检查 — 第 {N} 回合】

世界温度: {平静/正常/紧张}
上次事件: T{X} ({简述})
距上次事件: {Y} 轮
触发窗口: {频率范围}

当前场景: {current_scene}
在场角色: {列表}
活跃情节线: {列表}

══════════════════════
触发判定:
══════════════════════
□ 用户行动已产生重大后果？ → {是/否，是则跳过}
□ 冷却中？ → {是/否，是则跳过}
□ 高强度冲突中？ → {是/否，是则跳过}
□ 距上次事件达到触发窗口？ → {是/否}

如果触发，选择事件:
  事件类型: {环境变迁/NPC自主行动/情节回响/随机遭遇}
  事件强度: {轻/中/重}
  事件简述: {一句话}
  融入位置: 【世界回应】的环境描写中

如果不触发:
  跳过。无需在响应中提及。

⚠️ 生成的事件必须：与当前场景相容、与活跃情节线不矛盾、服务于叙事丰富度
```

## Stage 4B: 情绪传染与氛围漂移提示词

> 💡 **自动化**：以下计算可通过 `state-helper.js` 自动完成：
> ```js
> const { computeContagion } = require('./state-helper.js');
> const result = computeContagion({ characters, matrix, atmosphere, newShock });
> // → { updatedEmotions, updatedAtmosphere, contagionTrace }
> ```
> 优先调用 helper，仅在 helper 不可用时手动计算。

```
【情绪传染计算 — 第 {N} 回合】

══════════════════════
输入：当前情绪矩阵
══════════════════════
在场角色情绪:
  {角色A}: {情绪值} (基线: {基线值})
  {角色B}: {情绪值} (基线: {基线值})
  {角色C}: {情绪值} (基线: {基线值})

情绪关系:
  {角色A} → {角色B}: {定位}+{强度}
  {角色A} → {角色C}: {定位}+{强度}
  ...

══════════════════════
步骤 1: 场景情绪场
══════════════════════
社会影响力权重:
  {角色A}: ×{权重} ({领袖/普通人/边缘人})
  {角色B}: ×{权重}
场景情绪场 = Σ(情绪 × 权重) / 人数 = {值}

══════════════════════
步骤 2: 邻近传染
══════════════════════
{角色A}: 偏移={场景场 - 当前情绪} × {传染率} × {关系权重} = {结果}
  新情绪 = {旧情绪} + {偏移}
{角色B}: ...

══════════════════════
步骤 3: 情绪惯性回归
══════════════════════
{角色A}: 回归量 = ({基线} - {新情绪}) × 0.1 = {值}
  最终情绪 = {传染后情绪} + {回归量}

══════════════════════
步骤 4: 情绪矩阵交叉更新
══════════════════════
本回合互动:
  {角色A} 对 {角色B} 的互动: {简述} → 关系调整 {±N}
  ...

══════════════════════
氛围漂移
══════════════════════
环境基线: {值} ({地点})
在场情绪均值: {值}
事件冲击残余: {值} (来源: T{X} {事件简述}, 本轮衰减后残余)
→ 氛围值 = {基线} + {情绪均值} + {冲击残余} = {最终值} ({色调描述})

是否需注入新冲击？
□ 本回合有重大事件 → 注入冲击值 {±N}
□ 无 → 仅自然衰减（冲击残余 × 0.7）

══════════════════════
叙事影响
══════════════════════
氛围值 {最终值} → 【世界回应】环境描写色调: {色调关键词}
NPC 行为倾向: {描述}
角色 {角色名} 的情绪将影响其对话/行为: {具体表现}

⚠️ 以上计算过程不输出给用户——结果自然融入叙事
```

## Stage 8: 会话管理提示词

### 存档提示词

```
【存档 — 第 {N} 回合】

> 💡 使用 state-helper.js 存档：
> ```js
> const { saveState } = require('./state-helper.js');
> saveState('{sessionId}', { 当前完整状态块 });
> ```

存档成功后告知用户：已保存到 {sessionId}，第 {N} 回合。下次可通过"继续 {sessionId}"恢复。
```

### 续档提示词

```
【续档 — {sessionId}】

> 💡 使用 state-helper.js 读档：
> ```js
> const { loadState } = require('./state-helper.js');
> loadState('{sessionId}');  // 输出完整状态块到上下文
> ```

基于恢复的状态块，呈现前情提要和当前处境。询问用户：继续还是需要调整？
```
